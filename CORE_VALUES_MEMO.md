# Memo: Relon 核心价值与场景设计指南

## 1. 顶级定位：Logic as Portable Data

Relon 的载荷**就是可执行的逻辑**。它的存在意义是让业务逻辑——校验
规则、计费公式、工作流、风控策略——能够像 JSON 一样**写一次、存一
次、传一次**，被任何符合规范的 Relon 运行时**完全相同地**求值。

> 写一遍规则。存进数据库、配置文件、RPC 报文。从 Go、TypeScript、
> Swift、浏览器里去跑。任何地方拿到的答案都一样。

这是支配其它一切设计选择的**承重轴**：

* 同源 + 同输入 → 字节级一致的输出。无浮点歧义、无哈希迭代序泄漏、
  无环境变量隐式影响。
* 仓库内的 Rust 实现是**参考运行时**；语言**规范**与运行时无关。
* 所有跨进程边界的能力（FS、网络、原生函数）都必须由脚本**显式
  声明**、由宿主**显式授权**——不存在「隐式信任」逃生口。

---

## 2. 支撑性的三大能力维度

### A. 序列化与跨端对齐（Portability，承重轴的直接产出）
*   **本质：** 逻辑不被编译进二进制代码，像 JSON 一样可存储、可分发。
*   **痛点：** 同样的业务逻辑（计费规则、表单校验）要在 Go 后端、TS
    前端、Swift 移动端各写一遍，导致「逻辑漂移」。
*   **Relon 优势：** 逻辑一次编写，存入数据库或配置文件，多端解析器
    执行结果 100% 确定且一致。
*   **规范契约：** 标准库（`std/list`、`std/string`…）是规范的一部分；
    任何符合规范的运行时都必须按相同语义实现。脚本**不依赖**任何
    runtime-private 的全局名。

### B. 沙箱执行（Capability-Granted，确保跨端确定性的前提）
*   **本质：** 带计步器、零 IO 默认权限的纯函数 VM。
*   **痛点：** SaaS 平台需要让用户自定义规则。直接跑 JS 有风险，用
    JSON 描述逻辑又太笨重。
*   **Relon 优势：** 默认沙箱，性能极轻；执行步数、value 体积、原生
    函数白名单都由 `Capabilities` 显式控制。脚本**没有**绕开沙箱的
    fallback。

### C. 上下文感知引用（Context-Aware）
*   **本质：** `&prev`、`&parent`、`&root` 等语义化路径让数据具备
    「引力」。
*   **痛点：** 复杂嵌套配置中字段依赖邻居，在 JS 里手动维护 index
    和路径极易出错。
*   **Relon 优势：** 数据自带相对引用。无论片段被移动到哪里都能自动
    找到上下文并重算（类似 Excel 公式）——而且因为引用是声明式的，
    跨端执行结果仍然确定。

## 3. 业务案例库 (8 大杀手级场景)
1.  **Template (放大器):** 后端输出极简数据，Relon 渲染复杂 UI 描述。
2.  **Validation (IO 卸载):** 强契约校验，确保进入系统的数据逻辑自洽。
3.  **VM (SaaS 插件):** 零风险运行租户自定义规则。
4.  **Game (策划便捷):** 策划通过 Relon 片段描述多态关卡行为。
5.  **Sequential (自愈流水线):** 视频剪辑/工序排期，删除中间项后续自动对齐时间。
6.  **Portable (金融风控):** 跨端同步复杂的计算公式。
7.  **Reactive (自计算资产):** 嵌套层级的资产负债表自动响应。
8.  **Edge (边缘策略):** IoT 节点即时更新控制策略。

## 4. 技术基准实测 (Benchmark Insights)
通过今日的“极限压力测试”，我们验证了 Relon 作为嵌入式语言的顶级素质：

### A. 性能：微秒级响应与函数式增强
*   **简单求值:** **~12.7 μs** (Warm state)，每秒可处理 8 万次请求。
*   **重度逻辑:** 1000 次列表循环计算仅耗时 **~1.3 ms**。
*   **架构升级:** 实现了 **Core/Std 分离架构**。Core 层（Rust）提供高性能原语（如 `_list_map`, `_list_reduce`），Std 层（Relon）提供优雅的业务封装。支持原生代码回调 Relon 闭包，实现了真正的一等公民级函数式支持。

### B. 体积：跨越“MB 级”门槛
通过调整 IR 层优化策略（LTO + Opt-Z + Panic-Abort），我们实现了体积的断崖式下降：
*   **核心库 (Core):** **~589 KB** (降幅 > 50%)。
*   **完整 CLI:** **~898 KB** (首次破 1MB 大关)。
*   **WASM 预测:** 剥离 CLI 依赖后，预计可压减至 **250 KB** 左右，与 Lua 处于同一量级。

## 5. 结论与后续方向
Relon 已经证明了其在 **“高性能”**、**“函数式表达力”** 和 **“MB 以下足迹”** 之间的完美平衡。它是目前市场上最轻量、最安全的“可编程 JSON”方案。
