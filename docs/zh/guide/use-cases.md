# 业务场景与定位

> 这一页帮你判断**该不该把 Relon 引入你的项目**。先讲承重轴，再列
> 8 个典型杀手级场景；如果你的需求在这些里面就值得考虑，否则可能用
> 普通 JSON / 现有脚本语言更合适。

## 1. 顶级定位：Logic as Data

Relon 的载荷**就是可执行的逻辑**。它的存在意义是让业务逻辑——校验
规则、计费公式、工作流、风控策略——能够像 JSON 一样**写一次、存一
次、传一次**，由嵌入式的 Relon 运行时**确定地**求值。

这是支配其它一切设计选择的**承重轴**：

- 同源 + 同输入 → 字节级一致的输出。无浮点歧义、无哈希迭代序泄漏、
  无环境变量隐式影响。同一段 .relon 跑两次必然得到同一个结果，可
  重放、可 hash、可缓存。
- 所有跨进程边界的能力（FS、网络、原生函数）都必须由脚本**显式声
  明**、由宿主**显式授权**——不存在「隐式信任」逃生口。
- 业务逻辑是**数据**而不是二进制：可以存在数据库里热更新、可以放在
  RPC 报文里传输、可以审计 diff——不需要重新部署嵌入它的服务。

## 2. 三大支撑能力维度

### A. 逻辑作为可发布数据 (Logic-as-Data)

- **本质**：逻辑不被编译进二进制代码，像 JSON 一样可存储、可分发。
- **痛点**：业务规则改一次就要走「改代码 → 上线 → 灰度」的全套流
  程；规则散落在多个服务里维护，容易漂移。
- **Relon 优势**：逻辑一次编写，存入数据库或配置中心，运行时直接
  加载求值——业务规则的发布周期与服务部署解耦。多服务共享同一份
  .relon 库，类型契约由 `#schema` 兜底，不会在传输中走样。

### B. 沙箱执行 (Capability-Granted)

- **本质**：带计步器、零 IO 默认权限的纯函数 VM。
- **痛点**：SaaS 平台需要让用户自定义规则。直接跑 JS 有风险，用
  JSON 描述逻辑又太笨重。
- **Relon 优势**：默认沙箱，性能极轻；执行步数、value 体积、原生
  函数白名单都由 `Capabilities` 显式控制。脚本**没有**绕开沙箱的
  fallback。

详见 [沙箱与权限](./sandbox.md)。

### C. 上下文感知引用 (Context-Aware)

- **本质**：`&prev`、`&parent`、`&root` 等语义化路径让数据具备
  「引力」。
- **痛点**：复杂嵌套配置中字段依赖邻居，在 JS 里手动维护 index 和
  路径极易出错。
- **Relon 优势**：数据自带相对引用。无论片段被移动到哪里都能自动
  找到上下文并重算（类似 Excel 公式）——引用是声明式的，求值结果
  仍然确定。

## 3. 8 个典型杀手级场景

| # | 场景 | 一句话描述 |
|---|---|---|
| 1 | **Template (放大器)** | 后端输出极简数据，Relon 渲染复杂 UI 描述 |
| 2 | **Validation (IO 卸载)** | 强契约校验，确保进入系统的数据逻辑自洽 |
| 3 | **VM (SaaS 插件)** | 零风险运行租户自定义规则 |
| 4 | **Game (策划便捷)** | 策划通过 Relon 片段描述多态关卡行为 |
| 5 | **Sequential (自愈流水线)** | 视频剪辑 / 工序排期，删除中间项后续自动对齐时间 |
| 6 | **Hot-update (金融风控)** | 风控公式存在配置中心，改一行即生效，无需重启服务 |
| 7 | **Reactive (自计算资产)** | 嵌套层级的资产负债表自动响应 |
| 8 | **Edge (边缘策略)** | IoT 节点即时更新控制策略 |

## 4. 技术基准实测 (Benchmark Insights)

通过极限压力测试，Relon 作为嵌入式语言的素质已经验证：

### A. 性能：微秒级响应与函数式增强

- **简单求值**：~12.7 μs（warm state），每秒可处理 8 万次请求。
- **重度逻辑**：1000 次列表循环计算耗时 ~1.3 ms。
- **架构升级**：Core/Std 分离架构。Core 层（Rust）提供高性能原语
  （如 `_list_map`、`_list_reduce`），Std 层（Relon）提供优雅的业务
  封装。原生代码可回调 Relon 闭包，实现一等公民级函数式支持。

### B. 体积：跨越「MB 级」门槛

通过调整 IR 层优化策略（LTO + Opt-Z + Panic-Abort）：

- **核心库 (Core)**：~589 KB（降幅 > 50%）。
- **完整 CLI**：~898 KB（首次破 1 MB）。
- **WASM 预测**：剥离 CLI 依赖后预计可压缩至 ~250 KB，与 Lua 处于
  同一量级。

> 注：WASM 的实测体积仍待 CI 落地。

## 5. 结论

Relon 在 **「高性能」**、**「函数式表达力」** 和 **「MB 以下足迹」**
之间取得平衡。它面向的是「需要把业务逻辑当作数据来管理、并要求执行
确定可审计」的场景；如果你只需要静态配置或宿主语言原生脚本足以应付，
普通 JSON / Lua / JS 反而更合适。
